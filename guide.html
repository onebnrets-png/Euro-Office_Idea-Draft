1. Registration & Login

1.1 Registration

  1. Open your browser and navigate to https://euro-office.net
  2. You will see the Login screen with the Euro-Office logo, email and password fields
  3. Click "Register here" at the bottom of the login form
  4. Fill in your E-mail Address (must be valid) and Password (minimum 6 characters)
  5. Click "Register"
  6. Check your email inbox for a confirmation email and click the activation link
  7. Return to https://euro-office.net and log in with your credentials

1.2 Login

  1. Enter your E-mail Address and Password
  2. Click "Login"
  3. If 2FA is enabled, enter the verification code from your authenticator app
  4. You will be taken to the Home Screen

1.3 Language Selection

In the top-right corner, click SI or EN to switch the interface and AI generation language. The active language is highlighted in blue.

Every chapter logically flows from the previous one: Problem → Solution → Objectives → Activities → Results. EU evaluators check this chain for consistency. Euro-Office enforces this through AI cross-referencing.

3. API Key Configuration

⚠ IMPORTANT — DO THIS FIRST

Without a valid API key, AI generation will not work. You can still manually fill in all fields, but the AI-powered features (Fill, Enhance, Regenerate) require an API key.

3.1 Setting Up Your API Key

  1. Click the Settings gear icon in the top-right corner of the application
  2. In the Settings modal, select your AI Provider:
    Recommended Gemini (Google) — generous free tier, fast
    Alternative OpenRouter — multiple models, requires credits
    Alternative OpenAI — GPT models, requires paid key
  3. Paste your API Key in the key field
  4. Select the Model (e.g., gemini-2.5-flash-lite for Gemini)
  5. Click Save

3.2 Getting a Free Gemini API Key

  1. Go to https://aistudio.google.com/apikey
  2. Sign in with your Google account
  3. Click "Create API Key"
  4. Copy the key and paste it into Euro-Office Settings
✔ Free Tier

Gemini's free tier allows approximately 15 requests per minute, which is sufficient for generating all project sections. No credit card required.

4. Creating a Project

4.1 New Project

  1. From the Home screen, click the "+ New Project" button
  2. Enter a Project Title (working title — AI can refine it later)
  3. Select Language: EN (English) or SI (Slovenian)

  5. Choose Funding Model: Centralized or Decentralized

5. Problem Analysis

This is the foundation of your entire project. Every cause identified here must later be addressed by an activity, and every consequence must connect to an impact. Without a solid problem analysis, everything that follows is built on sand.

5.1 Core Problem

Enter a concise Title and a detailed Description with quantitative evidence. Even a few sentences are enough to start — AI will expand and enrich it with citations from real sources (Eurostat, OECD, European Commission, etc.).

5.2 Causes (minimum 4)

Click "+ Add Cause" to add manually, or let AI generate them. Causes should be ordered from root/structural causes first to proximate causes second. Each cause gets a title and a 3–5 sentence description with evidence-based citations.

5.3 Consequences (minimum 4)

Click "+ Add Consequence" or let AI generate them. Consequences should show the escalation chain: local impact → regional impact → national impact → EU-level policy impact.

5.4 AI Generation Modes

At the top of each section, you will find three generation buttons:

📝

Fill

Fills only empty fields. Keeps your existing content untouched.

✨

Enhance

Deepens existing content. Adds citations, EU terminology, and evidence.

🔄

Regenerate

Creates completely new content from scratch. Replaces existing data.

💡 Recommended Workflow

Write your core problem title and a few sentences of description manually. Then click Regenerate to let AI generate the full analysis with causes, consequences, and evidence. Review and edit the output, then use Enhance to further deepen the content with additional citations.

5.5 Inline Visualizations

After generation, a "Generate visualizations" button may appear below descriptions containing empirical data. Click it to create charts (bar charts, gauges, donuts) that visualize statistics mentioned in the text. These are useful for enriching your proposal and for the Dashboard overview.

6. Project Idea

This chapter translates the problem analysis into a proposed intervention. It contains multiple sub-sections, each accessible from the sidebar or by scrolling:

6.1 Project Title & Acronym

Title: A concise noun phrase (30–200 characters). Must NOT contain the acronym. Must NOT be a full sentence with a verb. Think of it as a project brand — concise, memorable, professional.

Acronym: 3–8 uppercase letters derived from the title keywords (e.g., GREENTRANS, DIGI-CRAFT, ALPSUST). Must be pronounceable or a recognisable abbreviation.

6.2 Main Aim

One comprehensive sentence starting with an infinitive verb (e.g., "To develop a cross-border digital platform that strengthens educational resilience..."). This captures the project's core purpose in a single powerful statement.

6.3 State of the Art

AI generates a thorough analysis referencing at least 3 real existing EU projects and studies, explicitly describing how your project capitalises on past results and avoids duplication (synergies and capitalisation — mandatory for EU proposals).

6.4 Proposed Solution

A detailed description of your proposed intervention. AI generates an introductory overview paragraph (5–8 sentences), followed by a phase-by-phase breakdown, and includes a mandatory DNSH (Do No Significant Harm) compliance statement confirming the project is screened against all six environmental objectives.

6.5 EU Policies

At least 3 relevant EU policies with specific alignment descriptions. Click "+ Add Policy" to add manually, or let AI generate them with real, verifiable policy references.

6.6 Readiness Levels

Four readiness assessments, each with a numerical level (1–9) and a project-specific justification:

LevelFull NameWhat it measures
TRLTechnology Readiness LevelTechnology maturity
SRLSocietal Readiness LevelSocial acceptance & adoption readiness
ORLOrganisational Readiness LevelInstitutional capacity for change
LRLLegislative Readiness LevelRegulatory and legal alignment

7. Objectives

7.1 General Objectives (3–5)

Each starts with an infinitive verb (e.g., "Strengthen educational resilience...", "Develop cross-border digital competences..."). These connect the project to broader EU goals. Each has a title, description (3–5 sentences), and a KPI indicator field.

7.2 Specific Objectives (5–7)

Follow the S.M.A.R.T. method: Specific, Measurable, Achievable, Relevant, Time-bound. Each has a measurable KPI indicator with quantitative targets, baseline references, and timeframes.

💡 Example S.M.A.R.T. Indicator

"Increase the percentage of pilot school students completing the academic year from 82% to 93% by M24, verified by school administration records across 5 partner regions."

Click "+ Add Objective" to add manually, or use the Fill/Enhance/Regenerate buttons for AI generation. Every indicator field must be filled — the AI enforces this as a quality gate.

8. Activities (Work Packages)

This is the most complex and content-rich section of the project. AI generates the full work plan in a two-phase approach.

8.1 Work Package Structure

AI generates 6–10 Work Packages in a mandatory order:

PositionWP TypeDuration
WP1Foundational / Analytical (incl. Capitalisation & Synergies task)Specific phase
WP2–WP(N-2)Content / Thematic WPs in logical sequenceSequential phases with overlaps
WP(N-1)Dissemination, Communication & ExploitationFull project duration (M1–end)
WP(N)Project Management & CoordinationFull project duration (M1–end)

8.2 Each Work Package Contains

Tasks (2–5) — each with title, description, start/end dates, and task dependencies (FS, SS, FF, SF types). Milestones (1–2) — completion checkpoints with specific dates. Deliverables (1–3) — with title, description, and a binary verifiable indicator that is Lump Sum compliant.

8.3 Two-Phase AI Generation

  1. Phase 1 — Scaffold: AI generates WP titles and date ranges for the entire project
  2. Phase 2 — Detail: AI generates each WP individually with full tasks, milestones, deliverables, and cross-WP dependencies

A progress indicator shows which WP is currently being generated. This process takes 1–3 minutes for 6–10 WPs — this is normal.

8.4 Editing Activities

Click on any WP to expand or collapse it. Edit tasks, milestones, and deliverables inline. Add or remove items with the + and trash buttons. Dates can be changed and the system automatically recalculates dependencies and enforces project timeframe boundaries.

💡 Temporal Integrity

Euro-Office automatically enforces that NO task, milestone, or deliverable falls outside the project start/end dates. If AI generates dates outside the envelope, they are automatically corrected.

9. Project Management

9.1 Management Description

AI generates a comprehensive narrative (500+ words) structured in clear paragraphs covering: governance structure with EU role abbreviations, decision-making mechanisms, quality assurance (including Gender Equality Plans and AI Act/GDPR compliance), risk management approach, internal communication tools and schedules, conflict resolution procedures, and data management with FAIR principles (referencing the mandatory DMP deliverable due by M6).

9.2 Management Structure (Organigram Labels)

Short labels that appear in the visual Organigram chart:

RoleEU AbbreviationExample Label
CoordinatorPK"Project Coordinator (PK)"
Steering CommitteeUO"Steering Committee (UO)"
Advisory BoardSO"Advisory Board (SO)"
WP LeadersVDS"WP Leaders (VDS)"
💡 Important

The structure fields are chart labels ONLY (max 5–8 words each). All detailed descriptions go in the main description field. Do not put long text in the structure fields.

10. Risk Register

AI generates 8–12 risks across four mandatory categories:

CategoryExamples
TechnicalTechnology failures, integration issues, data quality problems
SocialStakeholder resistance, low engagement, cultural barriers
EconomicBudget overruns, market changes, partner financial instability
EnvironmentalClimate events, regulatory changes, environmental compliance

Each risk includes: auto-generated ID (RISK1, RISK2...), category, title (noun phrase), description (2–4 sentences), likelihood (low/medium/high), impact (low/medium/high), and a concrete mitigation strategy with responsible parties and backup plans.

⚠ AI Act / GDPR

If your project involves digital or AI components, the risk register will automatically include an "Ethical and Regulatory Compliance" risk covering AI Act and GDPR requirements.

11. Expected Results & KERs

11.1 Outputs (5–8)

Direct, tangible deliverables of activities. Each has a result-oriented noun phrase title (e.g., "Digital Competence Curriculum" not "Develop a curriculum"), a 3–5 sentence description, and a binary verifiable indicator that is Lump Sum compliant.

11.2 Outcomes (4–6)

Medium-term changes resulting from outputs. Each has a measurable change indicator with quantitative targets and measurement methods. Clearly distinguishes Target Groups (engaged during project) from End Users (adopting results post-project).

11.3 Impacts (3–5)

Long-term strategic shifts linked to EU policy objectives. Each categorised into a Key Impact Pathway (KIP):

PathwayFocus Area
ScientificAdvancing knowledge, methods, standards
SocietalAddressing societal challenges, improving quality of life
EconomicInnovation, competitiveness, growth, employment

11.4 Key Exploitable Results — KERs (4–6)

Specific assets carrying lasting value beyond the project lifetime. Each has a specific product/asset name as title and an exploitation strategy covering the 3 sustainability pillars:

PillarQuestion it answers
FinancialHow will it be funded/maintained post-project?
InstitutionalWho takes ownership and operational responsibility?
Political/RegulatoryHow does it integrate into policy frameworks?
💡 Composite Generation

Clicking Regenerate on the Expected Results section generates Outputs, Outcomes, Impacts, and Risks together in one operation, ensuring cross-reference consistency across all result levels.

13. Finance (Budget Allocation)

13.1 Funding Model

At the top of the Finance section, you will see the selected funding model (Centralized or Decentralized). This was chosen when the project was created and determines how the budget is structured.

13.2 AI-Generated Partner Allocations

Click "Generate Partner Allocations with AI" to automatically allocate partners to tasks with realistic hours, person-months (1 PM = 143 hours, EU standard), and labour costs based on each partner's PM rate.

13.3 EU Budget Rules (Applied Automatically by AI)

The AI respects the following EU budget distribution rules when generating allocations:

Total Project BudgetMax % for Project Management WP
up to 500,000 EUR15%
500,001 – 1,000,000 EUR10%
1,000,001 – 3,000,000 EUR10%
3,000,001 – 5,000,000 EUR7%
5,000,001 – 10,000,000 EUR7%
over 10,000,000 EUR5%

The Dissemination, Communication & Exploitation WP receives approximately 15% of the total project budget (fixed for all project sizes). The remaining budget is distributed proportionally across content/thematic WPs.

15. Gantt Chart, PERT Chart & Organigram

15.1 Gantt Chart

A visual timeline of all work packages and tasks. Task durations are shown as horizontal bars, milestones as diamonds, and dependencies as connecting arrows. The chart is colour-coded by work package. Scroll horizontally to view the full project timeline. This chart updates automatically when you change task dates or add new activities.

15.2 PERT Chart

A network diagram showing task dependencies and the critical path. Each node represents a task with its duration. Arrows show the dependency flow between tasks (FS, SS, FF, SF). The critical path is highlighted, helping you identify bottlenecks and parallel workstreams. Particularly useful for project planning and resource optimization.

15.3 Organigram

An organisational chart displaying the project management structure: Coordinator → Steering Committee → Advisory Board → WP Leaders. The chart is generated from the labels defined in the Project Management section (Section 9). This is a standard visual required in most EU project proposals.

16. Export & Print

16.1 DOCX Export

  1. Click the Export / Download button in the top toolbar
  2. Select DOCX format
  3. A professional Word document is generated containing all chapters, formatted tables, structured headings, and properly ordered content
  4. The file downloads automatically to your computer

16.2 Print Layout

Use Ctrl+P (or Cmd+P on Mac) to access the print-optimized layout. The sidebar and navigation elements are automatically hidden. All content is formatted for clean A4 printing with proper page breaks between sections.

16.3 Project Summary

Click the Summary button to generate an AI-powered executive summary (max 800 words) covering: Project Overview, Problem & Need, Solution & Approach, Key Results & Impact, and EU Added Value. The summary follows the intervention logic golden thread and can be used as a project abstract.

17. Administration Panel

The Admin Panel is available to organisation administrators only. If you don't see it in your navigation, contact your organisation admin.

17.1 Admin Features

FeatureDescription
User ManagementView and manage users within your organisation
Knowledge BaseUpload reference documents (PDFs, Word files) that AI uses as mandatory context
AI Instructions OverrideCustomize AI generation rules for any section — quality gates, task instructions, etc.
Global InstructionsSet organisation-wide generation preferences and overrides
2FA ManagementEnable and configure Two-Factor Authentication for the organisation

17.2 Knowledge Base

  1. Go to Admin Panel → Knowledge Base
  2. Click "Upload Document"
  3. Select a PDF or Word file (e.g., call text, programme guide, previous project proposal, policy documents)
  4. The system extracts and stores the text content
  5. When any user in the organisation generates content, the AI will automatically consider these documents as authoritative reference material
💡 Pro Tip

Upload the actual call text or programme guide for your target funding programme. The AI will align generated content with the specific requirements, terminology, and evaluation criteria of that programme.

17.3 AI Instructions Override

Administrators can view and override any AI rule category: quality gates per section, task instructions, global rules, academic rigor requirements, humanization rules, and more. Overrides take effect immediately for all users in the organisation. Click "Reset to defaults" to restore original rules at any time.
