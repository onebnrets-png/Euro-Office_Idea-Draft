🚀 1. Getting Started

1.1 Registration

  1. Open your browser and navigate to https://euro-office.net
  2. You will see the Login screen with the Euro-Office logo, email and password fields
  3. Click "Register here" at the bottom of the login form
  4. Fill in your E-mail Address (must be valid) and Password (minimum 6 characters)
  5. Click "Register"
  6. Check your email inbox for a confirmation email and click the activation link
  7. Return to https://euro-office.net and log in with your credentials

1.2 Login

  1. Enter your E-mail Address and Password
  2. Click "Login"
  3. If 2FA is enabled, enter the verification code from your authenticator app
  4. You will be taken to the Home Screen / Dashboard

1.3 Language Selection

In the top-right corner, click SI or EN to switch the interface and AI generation language. The active language is highlighted in blue.

🔑 3. API Key Configuration

⚠ IMPORTANT — DO THIS FIRST

Without a valid API key, AI generation will not work. You can still manually fill in all fields, but the AI-powered features require an API key.

3.1 Setting Up Your API Key

  1. Click the Settings gear icon (⚙️) in the top-right corner
  2. In the Settings modal, select your AI Provider:
    Recommended Gemini (Google) — generous free tier
    Alternative OpenRouter — multiple models, requires credits
    Alternative OpenAI — GPT models, requires paid key
  3. Paste your API Key in the key field
  4. Select the Model (e.g., gemini-2.5-flash-lite)
  5. Click Save

3.2 Getting a Free Gemini API Key

  1. Go to https://aistudio.google.com/apikey
  2. Sign in with your Google account
  3. Click "Create API Key"
  4. Copy the key and paste it into Euro-Office Settings

🔍 5. Problem Analysis

This is the foundation of your entire project. Everything else builds on this chapter.

5.1 Core Problem

Enter a concise Title and a detailed Description with quantitative evidence. Even a few sentences are enough — AI will expand and enrich it.

5.2 Causes (minimum 4)

Click "+ Add Cause" to add manually, or let AI generate them. Causes should be ordered from root/structural to proximate. Each cause needs a title and description with evidence.

5.3 Consequences (minimum 4)

Click "+ Add Consequence" or let AI generate them. Consequences should show the escalation chain: local → regional → national → EU impact.

5.4 AI Generation

At the top of the section, you will find three generation buttons:

📝

Fill

Fills only empty fields. Keeps your existing content untouched.

✨

Enhance

Deepens and improves existing content. Adds citations and EU terminology.

🔄

Regenerate

Creates completely new content from scratch. Replaces existing data.

💡 Recommended Workflow

Write your core problem title and a few sentences of description manually. Then click Regenerate to let AI generate the full analysis. Review and edit, then use Enhance to further improve.

5.5 Inline Visualizations

After generation, a "Generate visualizations" button appears below descriptions containing empirical data. Click it to create charts (bar charts, gauges, donuts) that visualize statistics mentioned in the text.

💡 6. Project Idea

This chapter translates the problem into a proposed intervention. It contains multiple sub-sections:

6.1 Project Title & Acronym

Title: A concise noun phrase (30–200 characters), no acronym inside the title.
Acronym: 3–8 uppercase letters derived from the title (e.g., GREENTRANS, DIGI-CRAFT).

6.2 Main Aim

One comprehensive sentence starting with an infinitive verb (e.g., "To develop...", "To strengthen..."). Captures the project's core purpose.

6.3 State of the Art

Analysis of existing projects and studies. AI includes references to real EU projects and describes how your project capitalises on past results (synergies).

6.4 Proposed Solution

Detailed intervention description structured in phases. AI generates an introductory overview, phase breakdown, and DNSH compliance statement.

6.5 EU Policies

Relevant EU policies with alignment descriptions. Add manually or let AI generate them.

6.6 Readiness Levels

ORLOrganisational Readiness LevelInstitutional capacity
LRLLegislative Readiness LevelRegulatory alignment

🎯 7. Objectives

7.1 General Objectives (3–5)

Each starts with an infinitive verb (e.g., "Strengthen...", "Develop..."). Linked to broader EU goals. Each has a title, description, and KPI indicator.

7.2 Specific Objectives (5–7)

Follow the S.M.A.R.T. method: Specific, Measurable, Achievable, Relevant, Time-bound. Each has a measurable KPI with targets and timeframes.

💡 Example S.M.A.R.T. Indicator

"Increase the percentage of pilot school students completing the academic year from 82% to 93% by M24, verified by school records across 5 partner regions."

Click "+ Add Objective" to add manually, or use Fill/Enhance/Regenerate for AI generation.

⚙️ 8. Activities (Work Packages)

This is the most complex and content-rich section of the project.

8.1 Work Package Structure

AI generates 6–10 Work Packages in a mandatory order:

PositionWP TypeDuration
WP1Foundational / AnalyticalSpecific phase
WP2–WP(N-2)Content / ThematicSequential phases
WP(N-1)Dissemination, Communication & ExploitationFull project duration
WP(N)Project Management & CoordinationFull project duration

8.2 Each Work Package Contains

Tasks (2–5) with title, description, start/end dates, and dependencies. Milestones (1–2) marking completion checkpoints. Deliverables (1–3) with title, description, and binary verifiable indicator (Lump Sum compliant).

8.3 Two-Phase AI Generation

Activities generation uses a special approach:

  1. Phase 1 — Scaffold: AI generates WP titles and date ranges
  2. Phase 2 — Detail: AI generates each WP individually with full content

A progress indicator shows which WP is currently being generated. This process takes 1–3 minutes for 6–10 WPs.

8.4 Editing Activities

Click on any WP to expand/collapse it. Edit tasks, milestones, and deliverables inline. Add or remove items with the + and trash buttons. Dates can be changed and the system recalculates dependencies.

📈 11. Expected Results & KERs

11.1 Outputs (5–8)

Direct, tangible deliverables of activities. Each has a result-oriented title, description, and binary verifiable indicator.

11.2 Outcomes (4–6)

Medium-term changes resulting from outputs. Measurable indicators with targets and methods.

11.3 Impacts (3–5)

Long-term strategic shifts linked to EU policy. Each categorised into a Key Impact Pathway:

PathwayFocus
ScientificAdvancing knowledge, methods, standards
SocietalAddressing challenges, quality of life
EconomicInnovation, competitiveness, employment

11.4 Key Exploitable Results — KERs (4–6)

Assets carrying value beyond the project. Each exploitation strategy covers 3 sustainability pillars: Financial, Institutional, and Political/Regulatory sustainability.

💡 Composite Generation

Clicking Regenerate on Expected Results generates Outputs, Outcomes, Impacts, and Risks together, ensuring consistency across all result levels.

📅 15. Gantt Chart, PERT Chart & Organigram

15.1 Gantt Chart

Visual timeline of all work packages and tasks. Shows task durations as horizontal bars, milestones as diamonds, and dependencies as arrows. Colour-coded by work package. Scroll horizontally for the full project timeline.

15.2 PERT Chart

Network diagram showing task dependencies. Each node represents a task with its duration. Arrows show the dependency flow. Critical path is highlighted. Useful for identifying bottlenecks.

15.3 Organigram

Organisational chart showing the management structure: Coordinator → Steering Committee → Advisory Board → WP Leaders. Based on labels from the Project Management section.
